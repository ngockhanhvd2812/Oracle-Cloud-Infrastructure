Block Volume Migration

info

Định nghĩa

Block Volume trong OCI là dịch vụ lưu trữ block hiệu suất cao, có thể gắn vào các compute instance như một ổ cứng.
sync

Copy to another region

  • Sao chép Block Volume backup từ source region sang target region
  • Quá trình chuyển đổi asynchronous, không real-time
  • Khôi phục backup thành volume mới tại target region
  • Phù hợp cho disaster recovery và migration scenarios
sync_alt

Cross-region replication

  • Cross-region replication cho phép sao chép liên tục dữ liệu
  • Volume replica tại target region ở trạng thái không sử dụng được
  • Cần Activate the replica để tạo volume mới sử dụng được
  • RTO thấp - khôi phục nhanh khi sự cố
help_outline

Cross-region replication Q&A

Câu hỏi: Khi sử dụng cross-region replication, cần làm gì để tạo volume mới từ replica?
A. Không cần hành động, replica mặc định có sẵn (Sai)
B. Khởi tạo replica (Sai)
C. Kích hoạt replica (Đúng)
D. Trigger the replica (Sai)
Giải thích: Replica cần được Activate the replica để tạo volume mới sử dụng được.
stars

Lợi ích

  • Disaster Recovery - Sao lưu dữ liệu giữa các region
  • Migration - Di chuyển dữ liệu với downtime tối thiểu
  • Khả năng mở rộng - Triển khai ứng dụng đa region
  • Tự động hóa - Thiết lập một lần, tự động sao chép

Hybrid Cloud Extension (HCX) và OCVS

info

HCX - Giới thiệu

Hybrid Cloud Extension (HCX) là giải pháp di chuyển máy ảo (VM) từ môi trường on-premises sang Oracle Cloud VMware Solution (OCVS) một cách liền mạch.

  • HCX Manager - Trung tâm quản lý và điều phối
  • HCX Connector - Kết nối giữa on-premises và cloud
  • HCX Interconnect - Tạo kết nối mạng an toàn
  • HCX WAN Optimization - Tối ưu hóa truyền dữ liệu
compare_arrows

Các phương pháp Migration

  • HCX vMotion - Zero downtime cho business-critical VMs
  • HCX Bulk Migration - Minimal downtime cho non-critical VMs
  • HCX Cold Migration - High downtime cho dev/test VMs
  • HCX Replication Assisted vMotion (RAV) - Near-zero downtime cho large-scale
  • HCX OS Assisted Migration (OSAM) - Chọ KVM workloads
devices_other

KVM workloads với OSAM

HCX OS Assisted Migration (OSAM) cho phép migrate KVM workloads sang OCVS:

  • Chạy trên non-VMware platform
  • Cần validation trước cutover
  • Yêu cầu minimal downtime
  • Không migrate VMs vào OCVS qua OCM
cloud_queue

Oracle Cloud VMware Solution (OCVS)

OCVS là dịch vụ cho phép triển khai môi trường VMware SDDC trên OCI:

  • vSphere - Ảo hóa và quản lý máy ảo
  • vSAN - Lưu trữ định nghĩa bằng phần mềm
  • NSX-T - Mạng và bảo mật định nghĩa bằng phần mềm
  • HCX - Mở rộng hybrid cloud và di chuyển workload

OCI Database Migration service và Zero Downtime Migration (ZDM)

info

OCI Database Migration Service

OCI Database Migration service là dịch vụ được quản lý hoàn toàn giúp di chuyển databases từ on-premises hoặc các cloud khác sang Oracle Cloud Infrastructure.

  • Migration Console - Giao diện quản lý tập trung
  • Migration Agent - Cài đặt tại source database
  • Data Transfer Medium - Object Storage, File Storage, Database Link
  • Target Database - OCI Database, Autonomous Database
swap_horiz

Các phương pháp Transfer

Data Pump via Object Storage

Sử dụng Object Storage làm staging medium - phương pháp phổ biến nhất, hỗ trợ AWS RDS

Data Pump via File Storage

Sử dụng NFS File Storage làm trung gian - phù hợp cho môi trường có kết nối NFS

Data Pump via Database Link

Kết nối trực tiếp qua SQL*Net - phù hợp cho Standard Edition sources

Oracle Zero Downtime Migration (ZDM)

ZDM là công cụ migration của Oracle giúp di chuyển databases từ on-premises hoặc cloud khác sang Oracle Cloud với minimal hoặc zero downtime.

  • ZDM Service Host - Quản lý toàn bộ quy trình migration
  • ZDM Agent - Cài đặt tại source và target database
  • Data Transfer Components - Data Pump, GoldenGate, Data Guard
  • ZDM Repository - Lưu trữ metadata và trạng thái migration

Migration đến Autonomous Database

Autonomous Database (ADB) là dịch vụ database tự quản lý, tự bảo mật và tự sửa lỗi trên Oracle Cloud.

  • OCI Database Migration Service sử dụng ZDM engine với GoldenGate
  • Hỗ trợ online (continuous) migration đến ATP hoặc ADW
  • Data Pump thực hiện initial load
  • GoldenGate xử lý change data capture
  • Free GoldenGate license (183 days)

Load Balancer và Ingress Controller

Trong OKE, Load Balancer và Ingress Controller là các thành phần quan trọng giúp expose containerized applications ra bên ngoài cluster.

Load Balancer
  • Route traffic ở L4 (TCP/UDP)
  • Tạo OCI Load Balancer resource
  • Dành cho external access
Ingress Controller
  • Route traffic ở L7 (HTTP/S)
  • Sử dụng OCI native Ingress
  • Hỗ trợ host-based/path-based routing
